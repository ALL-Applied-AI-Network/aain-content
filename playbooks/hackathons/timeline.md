# Building Your Planning Timeline

This is an 8-week countdown for a mid-size hackathon (50-100 participants). If you have more time, spread out the early weeks. If you have less, compress weeks 8-6 but do not skip any steps.

## Week 8: Foundation

- [ ] Lock in format (see [Deciding Your Format](./format.md))
- [ ] Assemble your core organizing team (minimum 5 people for mid-size)
- [ ] Assign roles: logistics lead, sponsorship lead, marketing lead, tech lead, operations lead
- [ ] Set a target date (check the academic calendar for conflicts: exams, homecoming, breaks)
- [ ] Begin venue research (book early -- good spaces go fast)
- [ ] Create a shared project board (Notion, Trello, or GitHub Projects)
- [ ] Set a recurring weekly planning meeting (30 min max, keep it tight)

**Key milestone:** Date set, team assembled, venue shortlist ready.

## Week 7: Venue and Budget

- [ ] Confirm venue booking (get it in writing, confirm hours including setup/teardown)
- [ ] Verify venue specs: Wi-Fi capacity, number of power outlets, projector/screen, table layout
- [ ] Draft initial budget (see [Sponsorship and Budget](./sponsorship.md) for templates)
- [ ] Identify 10-15 potential sponsors to contact
- [ ] Begin drafting sponsorship packet
- [ ] Check if your university requires event insurance or safety plans

**Key milestone:** Venue confirmed, budget drafted.

## Week 6: Sponsorship Outreach and Registration Setup

- [ ] Send sponsorship emails (see [Sponsorship and Budget](./sponsorship.md) for templates)
- [ ] Set up registration platform (Luma, Eventbrite, or a Google Form)
- [ ] Write the registration page copy: what the hackathon is, who should attend, what to bring
- [ ] Design event branding (logo, banner, color scheme -- keep it simple)
- [ ] Open registration with an early-bird deadline 2 weeks out
- [ ] Follow up with sponsors who have not responded within 5 business days

**Key milestone:** Registration live, sponsorship outreach in progress.

## Week 5: Marketing Push

- [ ] Announce the event on all club channels (Discord, Instagram, email list)
- [ ] Post flyers in CS/engineering buildings (physical flyers still work)
- [ ] Ask professors to mention the event in relevant classes (give them a 2-sentence blurb)
- [ ] Cross-promote with related clubs (data science, robotics, business, design)
- [ ] Confirm first sponsor commitments and update budget accordingly
- [ ] Begin recruiting mentors (aim for 1 mentor per 5-8 participants)

**Key milestone:** Registration count at 30% of target, first sponsors confirmed.

## Week 4: Logistics Lock-In

- [ ] Finalize challenge tracks and judging criteria (see [Challenges and Judging](./challenges.md))
- [ ] Confirm mentor roster and send them a briefing doc (see [Mentorship and Support](./mentorship.md))
- [ ] Order event supplies: name badges, power strips, extension cords, markers, sticky notes
- [ ] Plan food: budget $8-12 per person per meal. Order from places that deliver in bulk.
- [ ] Set up the event Discord/Slack with channels: announcements, general, team-finding, help-desk, each challenge track
- [ ] Send a reminder email to all registrants with key details

**Tip:** For a Saturday noon to Sunday noon hackathon, plan for: lunch, dinner, midnight snacks, breakfast, and lunch again. That is 5 food runs. Budget accordingly.

**Key milestone:** Logistics plan complete, food ordered or vendors confirmed.

## Week 3: Content and Tech

- [ ] Finalize the event schedule (opening ceremony, hacking blocks, meals, demos, closing)
- [ ] Write the opening ceremony script (keep it under 15 minutes)
- [ ] Set up project submission platform (Devpost is standard, a Google Form works too)
- [ ] Test all A/V equipment in the venue
- [ ] Recruit and confirm judges (3-5 for a mid-size event)
- [ ] Send judges the rubric and logistics details
- [ ] Close early-bird registration, send a second marketing push

**Key milestone:** Schedule finalized, submission platform ready.

## Week 2: Final Prep

- [ ] Send a detailed participant email: schedule, what to bring, parking, team formation info
- [ ] Print signage: directional signs, Wi-Fi password, schedule poster, sponsor logos
- [ ] Confirm all food orders and delivery times
- [ ] Do a dry run of the opening ceremony and demo process
- [ ] Prepare prizes (order now if shipping is required)
- [ ] Create a day-of contact sheet for all organizers (phone numbers, roles)
- [ ] Close registration or manage the waitlist

**Key milestone:** Everything ordered, nothing left that requires shipping time.

## Week 1: Final Checks

- [ ] Send a final reminder email to participants (48 hours before)
- [ ] Charge all devices, pack all supplies, print all materials
- [ ] Hold a 15-minute organizer stand-up: everyone confirms their responsibilities
- [ ] Set up early if the venue allows it (the night before is ideal)
- [ ] Post a "what to bring" checklist on Discord: laptop, charger, water bottle, student ID
- [ ] Get some sleep. Seriously. You will need it.

**Key milestone:** Ready to go. Nothing left to do but show up.

## Compressed Timeline (3-4 Weeks)

Running a small club jam on a tight schedule? Here is the minimum viable timeline:

| Week | Focus |
|------|-------|
| 4 | Lock format, book room, open registration |
| 3 | Promote hard, recruit 2-3 mentors, plan food |
| 2 | Finalize schedule, set up Discord, order pizza |
| 1 | Final comms, setup, run the event |

You can skip formal sponsorship for a club jam. Budget $200-500 from club funds for food and small prizes.
